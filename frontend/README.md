# Emergency Call Centre Dashboard

Guardian Watch is a sophisticated, dark-themed dashboard built to monitor and triage emergency alerts generated by hardware buttons and ML audio listening devices installed in elderly residences.

## Features
- **Live Activity Monitoring**: A streaming alert queue sorted intuitively by severity tier (Urgent, Med, Low).
- **Signal Intelligence Engine**: Drill down into incoming calls to see precisely what hardware triggered it, ML pitch score confidence graphs, and language dialect detections (English, Mandarin, Hokkien, etc.).
- **False Positive Processing**: See exact validation criteria (Background Noise filtering, Accidental Extra Clicks thresholds).
- **Real-Time Data Layering**: Simulated via dummy WebSockets feeding straight into a centralized Redux state architecture allowing real-time state transitioning tracking (New -> Claimed -> Dispatched -> Resolved).

## Technologies Built With
- **React.js** (Vite Platform)
- **Tailwind CSS** + Native CSS custom variables for performance themes.
- **Redux Toolkit** (Global state management)
- **React Router v6** (Navigation flow & Authentication mocks)

## Backend Integration Schema

When swapping the internal `mockWebSocket.js` simulator for a production backend/socket stream, the engine expects the JSON payload for a single Alert Event to exactly match the shape below:

### `Alert Event` Payload Schema
```json
{
  "id": "string",                     // Unique identifier for the alert event
  "tier": "urgent" | "med" | "low",   // Severity classification
  "source": "btn" | "audio",          // Trigger type
  "keywords": ["string"],             // Array of transcoded keywords detected, e.g. ["வலிக்குது"]
  "languages": ["string"],            // Recognized dialect/language
  "location": "string",               // Human readable address format
  "coordinates": "string",            // Lat/Long string e.g. "1.3711°N, 103.8451°E"
  "pitch": number,                    // ML audio pitch score integer (0-100)
  "timeAgo": number,                  // Seconds since the original trigger capture
  "timestamp": "ISO Date String",     // Absolute timestamp of the trigger capture
  "actionState": "new" | "claimed" | "dispatched" | "resolved", // Current progressing state of the alert
  
  "metadata": {                       // Nested detail objects for Signal Intelligence panel
    "mlSource": "string",
    "pitchScoreDetail": "string",     // Explanation blurb for the pitch metric
    "dialectDetected": "string",
    "falsePositiveAnalysis": {        // Diagnostic metrics from edge sensors
      "extraClicks": { 
        "passed": boolean, 
        "desc": "string" 
      },
      "backgroundNoise": { 
        "passed": boolean, 
        "desc": "string" 
      },
      "silentAudio": { 
        "passed": boolean, 
        "desc": "string" 
      }
    }
  },
  
  "beneficiary": {                    // Data pulled from pab_beneficiaries SQL schema
    "nric": "string",
    "full_name": "string",
    "age": number,
    "phone_number": "string",
    "emergency_contact_name": "string",
    "emergency_contact": "string",
    "patient_medical_summary": "string",
    "DNR_status": "string",
    "primary_language": "string",
    "insurance_ward_class": "string",
    "primary_hospital": "string"
  }
}
```

## Integrating Live APIs/Sockets
1. Remove `startMockWebSocket` initialization in `App.jsx`.
2. Connect your secure `WebSocket` instance directly to your endpoint (e.g. `wss://api.guardian-watch.com/stream`).
3. Inside your socket's `onmessage` listener, map the incoming `event.data` to the schema and use `store.dispatch(addAlert(mappedData))` to organically feed the UI components without page tearing or refreshes.
